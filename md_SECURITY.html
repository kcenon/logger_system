Security Policy

Supported Versions

Version Supported
Latest :white_check_mark:
< Latest :x:

Only the latest release receives security updates. Users are encouraged to upgrade to the most recent version.

Reporting a Vulnerability

If you discover a security vulnerability in Logger System, please report it responsibly through one of the following channels:

  • Email: kcenon@naver.com
  • GitHub Security Advisories: Report here

Please do NOT report security vulnerabilities through public GitHub issues.

What to Include

  • Description of the vulnerability
  • Steps to reproduce the issue
  • Potential impact assessment
  • Suggested fix (if any)

Response Timeline

Stage Timeframe
Acknowledgment Within 7 days
Initial assessment Within 14 days
Fix and release Depends on severity

After Reporting

  1. You will receive an acknowledgment within 7 days
  2. The maintainers will investigate and provide an initial assessment
  3. A fix will be developed and tested
  4. A security advisory will be published with the fix release
  5. Credit will be given to the reporter (unless anonymity is requested)

Security Best Practices

When using Logger System in your projects:

  • Always use the latest stable release
  • Monitor the GitHub Security Advisories page for updates
