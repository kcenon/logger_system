Detailed Description

Tag interface for asynchronous writers.

Asynchronous writers use background processing for I/O. When write() returns, data is queued but may not be written yet.

Characteristics:

  • Non-blocking operation
  • Higher throughput potential
  • Requires explicit flush for guaranteed delivery
  • Suitable for high-volume logging

Examples: async_writer, batch_writer, network_writer
